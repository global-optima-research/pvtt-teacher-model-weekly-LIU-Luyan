## 一、项目全局概览

### 1.1 项目定义

PVTT 是一个**面向电商场景的产品视频智能编辑系统**。核心任务：

> 给定一段**源视频**（电商产品展示模板）、一张**参考产品图像**和一个**掩码**，系统输出一段将源视频中原产品**替换为参考产品**的高质量编辑视频。

用一句话概括：**把电商视频里的 A 商品替换成 B 商品，背景、运动、光照保持不变**。

### 1.2 技术路线总览（三阶段流水线）

│  │  数据集构建       │───>│  Teacher Model   │───>│  DMD 蒸馏加速    │  │

### 2.1 Task 1 整体目标

从电商平台采集原始视频素材，经过四阶段处理，产出 **20K+ 高质量训练三元组**。

### 2.2 数据集构建四阶段管线

```
阶段 1: 预处理
  ├── 电商视频采集 (Scrapy/Selenium, 500+ 产品视频)
  ├── 镜头检测与切割 (PySceneDetect ContentDetector)
  ├── 视频标准化 (720p, 24fps, 2-4s)
  └── 产品图片采集 (rembg/BiRefNet 去背景, 53+ 产品 × 多视角)

阶段 2: 分割与背景修复  ← 【刘麓琰负责】
  ├── 商品检测 (Grounded-SAM2 → BBox + 初始 Mask)
  ├── 视频级分割传播 (SAM2 video predictor → 全帧 Mask)
  ├── 分割质量检查 (IoU > 0.9, 边缘质量)
  ├── 背景修复 (VideoPainter → 纯背景视频)
  └── 交叉配对 (53 模板 × 52 产品 × 多视角 → ~35K 候选)

阶段 3: 视频合成
  ├── 路线 A: VideoAnyDoor / InsertAnywhere (主力合成)
  └── 路线 B: FLUX 编辑首帧 + Wan2.1 I2V (补充方案)

阶段 4: 质量过滤 (三级级联)
  ├── L1 硬性过滤: 完整性、BG SSIM > 0.85
  ├── L2 软性评分: CLIP-I, DINO-I, 时序一致性, MUSIQ
  └── L3 VLM + 人工: Qwen2-VL 语义评分 + 10% 人工抽检
```

## 三、刘麓琰负责部分 — 深度解析

### 3.1 工作定位

刘麓琰的工作位于**数据集构建管线的核心中间环节**，负责：
1. 从原始视频中**精确分割出商品区域**（生成 Mask 序列）
2. 对商品区域进行**视频修复**，获得纯背景视频
3. 将模板视频与产品图片进行**交叉配对**，生成训练候选对

这是整个数据管线中**技术难度最高的环节**，因为它直接决定了：
- Mask 质量 → 后续合成视频的边缘质量
- 背景修复质量 → 训练数据中背景的自然度
- 交叉配对策略 → 训练数据的多样性和规模

### 3.2 五个子任务详解

#### 子任务 1: 商品检测 (Grounded-SAM2)

```
输入: 电商产品视频的首帧
      ↓
Grounding DINO 1.5 Pro
  - 文本提示 (如 "handbag", "watch", "cosmetics")
  - 零样本检测, LVIS AP 55.7
      ↓
输出: 首帧中商品的 BBox + 初始 Mask
```

**关键技术**: Grounded-SAM2 = Grounding DINO (开放词汇检测) + SAM2 (分割)
- Grounding DINO 负责根据文本提示定位商品边界框
- SAM2 根据边界框生成精确的分割掩码

#### 子任务 2: 视频级商品分割 (SAM2 Video Predictor)

```
输入: 首帧 Mask + 整段视频
      ↓
SAM2 流式记忆架构
  - 图像编码器: MAE 预训练 Hiera (分层架构)
  - 记忆注意力: 自注意力 → 交叉注意力 (对记忆 + 对象指针)
  - 记忆库: FIFO 队列 (最近 N=6 帧) + 提示帧
  - 遮挡预测头: 处理商品被部分遮挡的场景
      ↓
输出: 每帧精确 Mask 序列 (全视频)
```

**核心原理**: SAM2 的视频传播流程：
1. 图像编码器处理当前帧
2. 记忆注意力将帧嵌入条件化于记忆库
3. 解码器输出当前帧的分割掩码
4. 记忆编码器将预测结果存入 FIFO 队列
5. 对象指针 token 用于后续帧的交叉注意力

**性能参考**: DAVIS 2017 val J&F 90.9-91.6, 推理速度 30-44 FPS

#### 子任务 3: 分割质量检查

| 检查项 | 标准 | 方法 |
|--------|------|------|
| Mask 时序一致性 | 相邻帧 IoU > 0.9 | 逐帧计算 IoU |
| 边缘质量 | 无明显锯齿/漏检 | 可视化检查 + 边缘平滑度指标 |
| 漏检/误检 | 商品完整覆盖 | 面积变化检测 (帧间 Δ < 20%) |

#### 子任务 4: 背景修复 (VideoPainter)

```
输入: 原始视频 + 商品 Mask 序列
      ↓
VideoPainter (SIGGRAPH 2025)
  双分支架构:
  ├── 上下文编码器: DiT 前 2 层 (仅 6% 参数)
  │   输入: 噪声潜变量 + 掩码视频潜变量 + 下采样掩码
  │   输出: 纯背景 token (前景 token 被排除)
  └── 冻结 DiT 骨干: 接收背景 token → 修复被遮挡区域
      ↓
输出: 纯背景视频 (商品区域被合理填充)
```

**关键特性**:
- **即插即用**: 可与 Wan2.1/2.2 DiT 骨干配合
- **任意长度**: 通过目标区域 ID 重采样支持长视频
- **性能**: DAVIS PSNR 25.27, SSIM 0.94

#### 子任务 5: 交叉配对

```
53 个模板视频 (已完成分割+背景修复)
  ×
52 个产品图片 (多视角 RGBA)
  ×
多视角变体
  ↓
~35K 候选训练对
  ↓
合理性过滤 (品类匹配、尺寸比例、视角兼容性)
```

## 四、刘麓琰工作与后续阶段的联系

### 4.1 对 Task 1 下游（王志铭）的影响

```
刘麓琰产出                         王志铭使用
─────────────                    ────────────
精确 Mask 序列  ───────────────>  VideoAnyDoor 合成时的运动轨迹
纯背景视频      ───────────────>  VideoAnyDoor 合成时的背景输入
~35K 候选对     ───────────────>  视频合成的输入配对
```

- **Mask 质量直接决定合成质量**: 如果 Mask 边缘不精确，VideoAnyDoor 合成的产品边缘会出现伪影
- **背景修复质量决定训练数据的背景自然度**: 如果修复有明显痕迹，模型会学到不自然的背景模式

### 4.2 对 Task 2 (Teacher Model 训练) 的影响

```
刘麓琰产出                         Task 2 使用
─────────────                    ────────────
Mask 序列       ───────────────>  训练数据中的 mask 条件输入
背景视频        ───────────────>  训练数据的 source_video 组成部分
配对策略        ───────────────>  训练数据多样性 → 模型泛化能力
```

Task 2 的 Teacher Model 架构 (Wan2.1-VACE-14B) 接收：
- **source video**: 即背景视频 + 新产品的合成结果
- **reference image**: 产品 RGBA 图片
- **mask**: 刘麓琰生成的商品分割 Mask

因此，Mask 的精度和背景修复的质量会**直接传导到 Teacher Model 的训练效果**。

### 4.3 对 Task 3 (DMD 蒸馏) 的间接影响

刘麓琰的工作通过数据质量**间接影响** Task 3:
- 高质量训练数据 → 高质量 Teacher Model → 蒸馏后的 Student Model 质量上限更高
- 精确的 Mask → 训练数据中的背景/前景分离更清晰 → 蒸馏时的背景保持损失 (L_bg) 更有效

## 五、项目涉及的核心技术背景

### 5.1 基础模型: Wan2.1/2.2

| 特性 | 说明 |
|------|------|
| 架构 | Diffusion Transformer (DiT) |
| 规模 | 1.3B / 14B 参数 |
| 3D VAE | 时空压缩比 4×8×8 |
| 注意力 | 14B: 3D Full Attention; 1.3B: 因子化时空注意力 |
| 编辑框架 | VACE (视频条件单元: 文本 + 上下文帧 + 时空掩码) |
| 训练范式 | Rectified Flow (流匹配) |

### 5.2 关键技术: 扩散模型视频编辑

项目处于 Video Editing 领域的最前沿，主要涉及：
- **DDIM Inversion / Rectified Flow**: 视频编辑的起点技术
- **注意力注入与操控**: Cross-attention 控制、Self-attention 注入
- **时间一致性**: Token 传播、光流引导、I2V 传播
- **ControlNet 适配**: 结构条件信号引导扩散过程

### 5.3 DMD 蒸馏核心思想

DMD 通过**分布级别的匹配**将多步扩散模型压缩为少步模型：
- DMD1 (CVPR 2024): 单步生成, ImageNet FID 2.62
- DMD2 (NeurIPS 2024): 消除回归损失, 集成 GAN 损失, FID 1.28
- 渐进式蒸馏: 50→16→8→4 步
